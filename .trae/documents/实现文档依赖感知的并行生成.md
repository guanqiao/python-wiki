## 问题分析

当前 `documentation_agent.py` 的 `_generate_all_docs` 方法将所有文档类型视为完全独立，使用 `asyncio.gather` 同时启动所有任务。这导致：

1. **没有定义文档间的依赖关系** - 某些文档需要其他文档的结果（如 `TECHNICAL_DESIGN_SPEC` 需要其他文档数据）
2. **没有依赖的文档本应并行，但缺少依赖感知调度** - 无法确保依赖文档先生成

## 解决方案

### 1. 在 `base.py` 中为 `DocType` 添加依赖定义

```python
class DocType(str, Enum):
    OVERVIEW = "overview"
    # ...
    
    @property
    def dependencies(self) -> list["DocType"]:
        """获取该文档类型的依赖"""
        DOC_DEPENDENCIES = {
            DocType.TECHNICAL_DESIGN_SPEC: [
                DocType.OVERVIEW,
                DocType.ARCHITECTURE,
                DocType.API,
                DocType.DEPENDENCIES,
            ],
            DocType.CODE_QUALITY: [DocType.MODULE],
            DocType.TEST_COVERAGE: [DocType.MODULE],
            # 其他文档类型没有依赖，可以并行
        }
        return DOC_DEPENDENCIES.get(self, [])
```

### 2. 修改 `documentation_agent.py` 的 `_generate_all_docs` 方法

使用类似 DAG 的调度策略：
- 第一层：没有依赖的文档（OVERVIEW, TECH_STACK, API, ARCHITECTURE, MODULE, DEPENDENCIES, CONFIGURATION, DEVELOPMENT, DATABASE 等）→ **并行执行**
- 第二层：依赖第一层的文档（TECHNICAL_DESIGN_SPEC, CODE_QUALITY, TEST_COVERAGE 等）→ **等依赖完成后并行执行**

核心逻辑：
```python
async def _generate_all_docs(...):
    # 1. 构建依赖图
    # 2. 拓扑排序分层
    # 3. 按层并行执行：
    #    - 每层内的文档并行执行
    #    - 层与层之间串行（等待上一层完成）
```

### 3. 具体修改文件

1. **`src/pywiki/generators/docs/base.py`** - 添加 `DOC_DEPENDENCIES` 配置
2. **`src/pywiki/agents/documentation_agent.py`** - 重写 `_generate_all_docs` 实现依赖感知调度

### 预期效果

- 无依赖的文档（如 OVERVIEW, TECH_STACK, API 等）将真正并行生成
- 有依赖的文档（如 TECHNICAL_DESIGN_SPEC）会等待其依赖文档完成后再生成
- 整体生成时间将显著缩短
