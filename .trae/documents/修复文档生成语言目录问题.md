## 问题原因

`documentation_agent.py` 中的 `_generate_all_docs` 和 `generate_single_doc` 方法在构建输出路径时存在 bug：
- `output_dir` 已经是完整路径
- 但代码把它传给 `WikiStructureConfig.base_dir`，然后 `get_wiki_root()` 又将其作为相对路径拼接到 `project_path`
- 导致语言子目录（`zh`/`en`）被添加到错误的路径上

## 修复方案

修改 `documentation_agent.py` 中的两处路径构建逻辑：

### 1. `_generate_all_docs` 方法（第 253-262 行）
将：
```python
structure_manager = WikiStructureManager(WikiStructureConfig(
    base_dir=str(output_dir),
    multi_language=True,
    default_language=language,
))
output_path = structure_manager.get_wiki_root(context.project_path or Path("."), language)
```
改为：
```python
output_path = output_dir / language.value
```

### 2. `generate_single_doc` 方法（第 350-362 行）
将：
```python
structure_manager = WikiStructureManager(WikiStructureConfig(
    base_dir=str(output_dir),
    multi_language=True,
    default_language=language,
))
output_path = structure_manager.get_wiki_root(context.project_path or Path("."), language)
```
改为：
```python
output_path = output_dir / language.value
```

这样可以确保文档正确生成到 `.python-wiki/repowiki/zh/` 和 `.python-wiki/repowiki/en/` 目录。
