## 修复计划

### 问题根源
数据库文档生成器在解析 SQL 文件时存在以下问题：
1. 正则表达式无法正确处理嵌套括号（如 CHECK、FOREIGN KEY 约束）
2. 列解析使用简单逗号分割，无法处理复杂约束
3. 字段类型被截断（如 `VARCHAR2(120` 而非 `VARCHAR2(120)`）
4. E-R 图字段数量限制过严

### 修复方案

#### 1. 改进 SQL 文件解析 (`_extract_from_sql_files`)
- 使用更智能的括号匹配算法，正确处理嵌套括号
- 改进列定义解析，正确识别完整的字段类型
- 处理 PRIMARY KEY、FOREIGN KEY、UNIQUE 等表级约束

#### 2. 增强 E-R 图生成 (`_generate_er_diagram`)
- 增加显示的字段数量（从6个增加到10个）
- 增加显示的表数量（从15个增加到25个）
- 优先显示主键和外键字段

#### 3. 改进字段类型解析
- 正确提取完整的字段类型（包括长度、精度）
- 处理各种数据库的类型语法差异

#### 4. 添加表去重逻辑
- 确保同一个表不会被重复添加

### 修改文件
- `src/pywiki/generators/docs/database_generator.py`

### 预期效果
- 完整解析所有表字段
- 正确显示字段类型
- 生成更完整的 E-R 图
- 避免表重复
