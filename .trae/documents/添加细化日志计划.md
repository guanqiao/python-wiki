## 日志增强计划

根据代码分析，将在以下关键位置添加细化日志，特别是耗时操作：

### 1. LLM 客户端 (`llm/client.py`)
- 将 `generate()` 和 `agenerate()` 的日志级别从 debug 改为 info
- 为 `stream()` 和 `astream()` 添加耗时日志
- 为 `test_connection()` 添加耗时日志
- 添加 token 统计信息

### 2. 文档生成 Agent (`agents/documentation_agent.py`)
- `_generate_single_doc()`: 添加单文档生成的开始/完成耗时日志
- `_generate_all_docs()`: 添加并发统计和各阶段耗时
- `generate_single_doc()`: 添加耗时日志
- `generate_incremental()`: 添加增量生成的各阶段耗时

### 3. 文档生成器基类 (`generators/docs/base.py`)
- `generate_with_llm()`: 添加 LLM 增强的耗时日志

### 4. Overview 生成器 (`generators/docs/overview_generator.py`)
- `_enhance_with_llm()`: 添加 LLM 增强的耗时日志和 prompt 长度

### 5. 向量存储 (`knowledge/vector_store.py`)
- `add_document()`: 添加文档添加耗时和分块统计
- `add_documents()`: 添加批量添加耗时
- `search()`: 添加搜索耗时
- `search_with_score()`: 添加带分数搜索耗时
- `_initialize_store()`: 添加初始化耗时
- `_save_store()`: 添加保存耗时

### 6. 搜索引擎 (`search/engine.py`)
- `index_project()`: 添加索引耗时和文档统计
- `index_module()`: 添加模块索引耗时
- `index_file()`: 添加文件索引耗时
- `search()`: 添加缓存命中/未命中日志

### 7. Wiki 导出 (`wiki/export.py`)
- `export_markdown()`: 添加导出耗时和文件统计
- `export_html()`: 添加 HTML 转换耗时和文件统计
- `export_pdf()`: 添加 PDF 生成耗时
- `export_all()`: 添加各格式导出的耗时统计

### 日志格式规范
```python
# 开始日志
logger.info(f"开始{操作名称}: 参数信息...")

# 完成日志
duration_ms = (time.time() - start_time) * 1000
logger.info(f"{操作名称}完成: 耗时={duration_ms:.0f}ms, 结果统计...")

# 过程日志（耗时操作中间步骤）
logger.info(f"{操作名称}阶段进度: 已处理 X/Y, 当前项={item}")
```
