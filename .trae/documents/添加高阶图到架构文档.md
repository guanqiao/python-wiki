## 核心思想
所有架构图都通过 LLM 进行高阶抽象，从代码细节中提取业务概念和架构模式，生成高层次的架构视图。

## 架构图分类与抽象策略

### 1. 系统全景图（现有）
- **输入**: 项目模块结构、依赖关系
- **LLM抽象**: 识别系统边界、子系统划分、核心职责
- **输出**: 高层次系统架构图

### 2. C4 模型图（现有）
- **Context**: LLM识别外部系统交互、用户角色
- **Container**: LLM抽象应用/服务/数据存储容器
- **Component**: LLM识别业务组件而非技术组件

### 3. 五种高阶图（新增）
- **流程图**: LLM抽象业务流程、系统流程
- **序列图**: LLM识别关键交互场景、时序关系
- **类图**: LLM提取领域模型、实体关系
- **状态图**: LLM识别业务状态、生命周期
- **组件图**: LLM抽象组件边界、依赖关系

### 4. 数据流图（现有）
- LLM抽象数据流转、处理节点

## 实现步骤

### Phase 1: 重构现有图表生成
1. 为现有图表添加 LLM 抽象层
2. 统一 LLM 调用接口
3. 设计标准化的抽象提示词

### Phase 2: 添加高阶图
1. 集成五种图表生成器
2. 实现 LLM 驱动的抽象方法
3. 更新模板展示

### Phase 3: 统一架构
1. 建立图表生成管道
2. 缓存 LLM 结果
3. 优化性能和成本

## 技术方案
- 使用统一的 `_abstract_with_llm()` 方法
- 为每种图表类型设计专门的 prompt
- LLM 返回标准化的图表数据结构
- 图表生成器负责渲染为 Mermaid
