## 优化方案

### 1. 扩展 `DocGeneratorContext` 添加项目语言字段
- 在 `base.py` 中添加 `project_language: str = "python"` 字段

### 2. 修改 `development_generator.py` 支持多语言
- **前置条件检测**：
  - Java: 检测 `pom.xml` 或 `build.gradle` 获取 JDK 版本
  - TypeScript: 检测 `package.json` 获取 Node.js 版本
  - Python: 保持现有逻辑
  
- **安装命令**：
  - Java: `mvn install` 或 `gradle build`
  - TypeScript: `npm install` 或 `yarn`
  - Python: 保持现有逻辑

- **测试命令**：
  - Java: `mvn test` 或 `gradle test`
  - TypeScript: `npm test`
  - Python: 保持现有逻辑

### 3. 修改 `config_generator.py` 支持多语言
- **系统要求**：根据项目语言设置默认值
- **配置文件检测**：
  - Java: `pom.xml`, `build.gradle`, `application.properties`, `application.yml`
  - TypeScript: `package.json`, `tsconfig.json`, `.eslintrc.js`
  - Python: 保持现有逻辑

### 4. 修改 `overview_generator.py` 技术栈分类
- 添加 Java 和 TypeScript 的常用库分类

### 5. 添加项目语言检测工具函数
- 在 `base.py` 中添加 `_detect_project_language()` 方法
